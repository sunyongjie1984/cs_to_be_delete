回复：VC6对STL容器支持度不佳

为了弄清楚这个问题，我翻看了Effective STL的附录B：对Microsoft的STL平台的说明，才搞懂了这个问题

其实这是历史原因。在VC4和VC5中，这样的代码是不合法的：
...
class Widget
{
    ...
    template <typename T>
        void dosomething(T item);
    ...
};
...
也就是说，它不支持模板成员函数。于是我们想要调用的构造函数：
template <typename It>
Deque(It first, It last)
实际上是不可能存在于VC4和VC5中的
不幸的是，VC6使用的STL库最初是为VC4开发的，而该版本的编译器缺乏对模板成员函数的支持――虽然当时几乎所有的C++编译器都没有这样的特性。后来VC6增加了这个特性，可是MS居然忘记了STL，唔……

D:\vcprojects\1234.cpp(11) : error C2440: 'initializing' : cannot convert from 'class std::_Tree<class std::string,struct std::pair<class std::string const ,class std::string >,struct Map::_Kfn,struct std::less<class std::string >,class std::allocator<class std::string > >::const_iterator' to 'class std::_Tree<class std::string,struct std::pair<class std::string const ,class std::string >,struct Map::_Kfn,struct std::less<class std::string >,class std::allocator<class std::string > >::iterator'
这里我们就可以清楚地看出，出问题的是一个名为_Tree的模板。在我们的印象中，开头一个下划线紧跟一个大写字母的名称一般是留给库的实现者使用的。可是我们对它一无所知。于是我们干脆一不做二不休，把传递给_Tree的模板参数全部替换为SOMETHING，看看会有什么样的效果：

回复：VC6对STL容器支持度不佳

但是我们还不怎么明白。这个错误信息唯一的可能是，我们试图用cont的find成员函数返回的const_iterator来初始化i。但是我们声明cont的时候并没有按const声明，find应该返回一个非const的iterator才对。
真的是这样吗？
再仔细看一下。cont确实是一个非const的map，但是dosomething却是一个const成员函数。它对cont进行操作的时候所有的非静态成员都会被转换为相应的const类型。所以解决方法有两个：一个是把i声明为const_iterator，另一个是把dosomething声明为一个非const成员函数。而这两种办法都比读懂那冗长晦涩的错误消息要简单得多。

另求教高人读懂VC6在Debug模式下使用map时出现的警告信息，附上源码：
#include <map>

int main()
{
    map<int, int> a;
    return 0;
}
